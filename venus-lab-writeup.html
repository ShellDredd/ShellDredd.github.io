Nivel [01] -- HACKER

mission.txt

## ES ##
La usuaria sophia ha guardado su contraseña en un fichero oculto en esta carpeta.Encuentralo y logueate como sophia.


Solución

Para resolver este reto necesitamos conocer las órdenes de sistema correspondientes al listado y lectura de archivos y directorios ocultos. Disponemos de variantes para estos dos usos, veámos unas cuantas.

# Listar archivos o directorios ocultos:

~$ ls -la
~$ ls -al
~$ ls -a


#Lectura de archivos:
---Estos son ejemplos de comandos de lectura y tienen diversos parámetros para diferentes usos, no dudes en revisarlos usando el manual de comandos de sistema: "man cat | man head | man more"---

~$ cat .myhiddenpazz
~$ less mission.txt
~$ more .myhiddenpazz
~$ head -n 70 archivo #(en este caso con head podemos indicarle que lea las 70 primeras líneas de un archivo).


Notas Adicionales:

Ahora que conocemos la usuaria y su contraseña, procedemos a loguearnos en el sistema usando los comandos permitidos por el mismo para esta función.


~$ su sophia

 El directorio actual en el que nos encontramos es /pwned/hacker/ y necesitamos movernos al directorio de la usuaria SOPHIA -- /pwned/sophia/

# Ver el directorio actual en el que nos encontramos:

~$ pwd

 # Cambiar o moverse de directorio en el sistema:

~$ cd /pwned/sophia/
~$ cd /cualquier/directorio/

 # En este directorio encontramos la primera flag para introducir en la plataforma y ganar puntos en el ranking.

~$ head -n 1 flagz.txt

 # También podemos ver otros archivos que podemos leer y que serán comunes en todo el laboratorio de Venus. Como hemos visto en el directorio del usuario hacker y en este de sophia, tenemos dos archivos comunes entre usuari@s, el "mission.txt" que nos revela las pistas para pivotar al siguiente usuari@ y nivel, y el de "flagz.txt" donde se encuentra la flag para introducir en la plataforma.

~$ ls -la
~$ cat mission.txt

 # Un hábito bueno para el listado de archivos y directorios, es el uso del parámetro "R" en la orden de sistema "ls -la" que permite el listado de archivos de forma recursiva en los directorios dentro del directorio actual.

~$ ls -laR

 # O en las rutas de directorios que le indiquemos:

~$ ls -laR /usr/share/wordlist/



Nivel [02] -- SOPHIA

mission.txt

## ES ##
La usuaria angela ha guardado su password en un fichero pero no recuerda donde... solo recuerda que el fichero se llamaba whereismypazz.txt


Solución

Para resolver este reto necesitamos conocer las órdenes de sistema correspondientes a la búsqueda de directorios y archivos en todo el sistema.
Existe un comando muy útil y con una parametrización amplia destinada a la búsqueda en sistema, lo usaremos para este caso.

# El parámetro "/" indica el directorio raiz de donde partirá la búsqueda, el "-name" para la entrada del nombre de archivo y "2>/dev/null" redirige el stderr (2) hacia el directorio /dev/null que podríamos decir que es como el agugero negro del sistema, la papelera del sistama.:

~$ find / -name whereismypazz.txt 2>/dev/null

 # El stdout o salida del comando nos indica que el archivo se encuentra en /usr/share/whereismypazz.txt así que procedemos a leer el contenido del mismo:
---Reordemos que existen rutas relativas y absolutas, además no necesitamos movernos al directorio de un archivo para manipularlo, podemos hacerlo usando la ruta completa.---

~$ cat /usr/share/whereismypazz.txt


Notas Adicionales:

Ahora que conocemos la usuaria y su contraseña, procedemos a loguearnos en el sistema usando los comandos permitidos por el mismo para esta función y añadiendo un guión "-" para hacer el login de usuari@ directamente a su directorio personal.


~$ su - angela

Nivel [03] -- ANGELA

mission.txt

## ES ##
La password de la usuaria emma esta en la linea 4069 del fichero findme.txt


Solución

Para resolver este reto necesitamos conocer las órdenes de sistema correspondientes a la lectura de archivos y cómo formatear el stdout o salida del comando con filtros de sintaxis regular para conseguir el resultado óptimo.
Para este caso utilizaremos un filtro sobre el output del comando.

# Primero con una tubería o pipe el cual se define añadiendo: " | " y que nos permite enlazar el filtro "awk" destinado para formatear output, sobre el que también añadiremos la sintáxis "'NR==4096'" consiguiendo mostrar por el output de pantalla exclusivamente la línea 4096 del archivo de texto "findme.txt" que encontramos en el directorio de la usuaria ANGELA.

~$ cat findme.txt | awk 'NR==4096'


Notas Adicionales:

Ahora que conocemos la usuaria y su contraseña, procedemos a loguearnos en el sistema usando los comandos permitidos por el mismo para esta función.


~$ su emma

 El directorio actual en el que nos encontramos es /pwned/angela/ y necesitamos movernos al directorio de la usuaria EMMA -- /pwned/emma/

# Ver el directorio actual en el que nos encontramos:

~$ pwd

# Cambiar o moverse de directorio en sistema:

~$ cd /pwned/angela/

Nivel [04] -- EMMA

mission.txt

La usuaria mia ha dejado su password en el fichero -.


Solución

Para resolver este reto es necesario conocer comandos de lectura de archivos y en el caso de "cat" se necesita añadir "./" para que no lo interprete como un atributo el guión.


# A continuación tendrás ejemplos varios de comandos de lectura:

~$ cat ./-
~$ more -
~$ head -n 1 -

# También podemos leer el archivo haciendo uso de las rutas completas de directorios donde se ubica el mismo.

~$ cat $PWD/-
~$ cat /pwned/emma/-
~$ more /pwned/emma/-
~$ cat < -


Notas Adicionales:

Ahora que conocemos la usuaria y su contraseña, procedemos a loguearnos en el sistema usando los comandos permitidos por el mismo para esta función.


~$ su - mia

Nivel [05] -- MIA

mission.txt

Parece que la usuaria camila ha dejado su password dentro de una carpeta llamada hereiam.


Solución

Para resolver este reto es necesario buscar un directorio en el sistema y para ello utilizaremos la orden de sistema "find" con dos parámetros, "-type d" que indica que buscamos un tipo directorio y con "-name" especificamos el nombre del archivo.


# Comandos de ejemplo:

~$ find / -type d -name archivo 2>/dev/null
~$ find / -name archivo 2>/dev/null

# A continuación recibimos el output con el directorio y hacemos una lectura de directorios y archivos sobre el mismo:

~$ ls -laR /opt/hereiam

# Observamos un archivo llamado ".here" y procedemos a su lectura.

~$ more /opt/hereiam/.here
~$ cat /opt/hereiam/.here


Notas Adicionales:

Las partes omitidas de explicación se contemplan en las notas adicionales de los primeros retos, échale un ojo.
Ahora que conocemos la usuaria y su contraseña, procedemos a loguearnos en el sistema usando los comandos permitidos por el mismo para esta función.


~$ su - camila



Nivel [06] -- CAMILA

mission.txt

La usuaria luna ha dejado su password en algun fichero dentro de la carpeta muack.


Solución

# Comandos de lectura y listado de directorios y archivos en el directorio /pwned/camila:



~$ ls -la
~$ ls -laR muack

Para resolver este reto es necesario buscar una diferencia entre todos los archivos y directorios que se encuentran alojados en el directorio MUACK y para ello utilizaremos la orden de sistema "find" y varios parámetros así como tuberías y filtros para poder leer directamente el archivo deseado ya que, el archivo no se puede abrir para lectura de forma normal.


# Utilizamos el parámetro "-type f" para buscar un tipo "file" de archivo y añadimos la orden "ls -la" para que liste y muestre atributos del archivo encontrado

~$ find muack -type f -ls

# Sabiendo que es un tipo de archivo podemos identificar qué tipo de file es el mismo con:

~$ find muck -type f | xargs file

# Procedemos a la lectura del archivo y nos encontramos con que no podemos leer con un simple cat o more, utilizaremos parametrización y argumentos de sintaxis regular en la propia orden de sistema "find"
"xargs" es un comando en Unix y la mayoría de los sistemas operativos similares a Unix que se utilizan para construir y ejecutar comandos desde la entrada estándar. Convierte la entrada de la entrada estándar en argumentos a un comando.
"-exec" nos permite ejecutar un proceso almacenado y "{}" es el que define y marca ese proceso que sabemos que es un tipo file por lo que usamos un comando de lectura combinado con lo anterior "-exec cat {} \;"
~$ find muack -type f 2>/dev/null | xargs cat
~$ find mucak/ -type f -exec cat {} \;
 # "xargs" es un comando en Unix y la mayoría de los sistemas operativos similares a Unix que se utilizan para construir y ejecutar comandos desde la entrada estándar. Convierte la entrada de la entrada estándar en argumentos a un comando.


Notas Adicionales:

Las partes omitidas de explicación como por ejemplo el uso de "2>/dev/null" , se contemplan en las notas adicionales de los primeros retos, échale un ojo.
Ahora que conocemos la usuaria y su contraseña, procedemos a loguearnos en el sistema usando los comandos permitidos por el mismo para esta función.


~$ su - luna
